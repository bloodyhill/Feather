## 0.2.7

### Fixed

- **Stale settings on object-cache hosts.** Toggled optimizations could briefly show as "not applied" on sites running Redis, Memcached, or LiteSpeed object caches. The settings repository now busts the WordPress options cache (`feather_settings` and `alloptions`) explicitly after every save, so follow-up requests read the canonical database value immediately.
- **Page caches holding REST responses.** Site Scan results and the "Take measurement" output could appear stale until a full page reload on sites with LiteSpeed Cache, WP Rocket, or Cloudflare. Every `feather/v1` REST response is now stamped with strict `Cache-Control: no-store` headers so page caches stop holding onto authenticated responses.

### Changed

- In-WordPress plugin name shortened to **Feather** so the Plugins list and Updates screen show a clean brand name. The longer descriptive title remains on the WordPress.org listing for search visibility.
