## 0.2.8

### Compatibility

- **WordPress 7.0 and Elementor 4.0.9.** Tested against the WP 7.0 admin (DataViews, refreshed admin chrome) and the now-default Elementor Atomic Editor.
- **Atomic element coverage.** Scanner recognises `e-self-hosted-video` and the standalone `e-video` atomic element, plus the existing `e-tabs` / `e-youtube` / `e-heading` / `e-button` / `e-paragraph` / `e-image` / `e-svg` / `e-component` set.

### Added

- **Block Elementor telemetry — Dashboard switch.** A single toggle bundles three previously-separate feature ids (`f.elementor.telemetry`, `f.elementor.api_fetcher`, `f.elementor.ai_module`) so the most common privacy/perf wins are one click. Granular toggles remain on the Features page.
- **"Edit with Elementor" admin bar item — removable.** New optimizer `f.elementor.admin_bar_edit_link` drops the floating link Elementor adds to the WP admin bar. Post-edit screen buttons still open the editor.
- **Background auto-rescan on save.** When an Elementor page is saved or published, Feather refreshes that single post's scan row in the background so gated optimizations stay accurate. Per-post throttle of 60 seconds; the existing cron tick handles the work.
- **Per-page feature overrides.** A sidebar meta box on the post edit screen disables individual Feather features on a single page without touching the sitewide setting. Wired into the frontend asset gate, JS defer, per-page asset trim, iframe lazy-load, image dimensions, and below-fold rendering.
- **Settings export / import.** Download your Feather configuration as a JSON file from Settings → Portability, and re-import on another site. Scan history and metrics history are not included in the export.
- **Core Web Vitals tiles on the Dashboard.** The page-weight probe now captures response time, a TTFB estimate, and a HEAD-probed hero-image size for same-origin images. New tiles appear on the Dashboard whenever a measurement includes these signals.
- **Elementor 4.0.x Global Classes detection** in the site scan. A new `uses_global_classes` flag and a sitewide `global_classes_usage_count` aggregate let gated optimizers refuse to strip handles that a Global Class references.
- **Database cleanup tools** for spam comments and auto-draft posts. Batched in groups of 200 per click.
- **Per-context heartbeat tuning.** The WP Heartbeat optimizer now exposes separate intervals for the block editor and other admin pages, with the public-frontend disable still default-on. Settings live under `advanced.heartbeat`.

### Changed

- **Pause-all card copy clarified.** The toggle now reads "Pause all optimizations for editing", and the paused-state primary button reads "Resume optimizations", so the workflow is unambiguous when you flip Feather off to edit in the builder.
- **Brand mark renders the correct variant on first paint.** Previously the topbar could briefly show the cream logo on a light theme (and vice versa) when the OS color preference disagreed with the saved Feather theme. The theme is now applied synchronously before the first render and passed explicitly to the mark.
- **Consolidated Elementor experiments feature.** `f.elementor.force_experiments`, `f.elementor.force_extra_experiments`, and `f.elementor.element_cache` are now one feature card, "Force-enable Elementor performance experiments". The old IDs migrate automatically — sub-toggles for the extra experiments and element cache live under `advanced.experiments`. Existing on/off state carries over.

### Internal

- Schema version bumped from 1 → 2; a one-time migration consolidates the legacy experiment feature ids and seeds the new `advanced` section in the settings option.
